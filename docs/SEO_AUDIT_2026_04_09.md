### C-2. `public/logo.png` 파일 없음 — 스키마 logo 404
**심각도:** CRITICAL | **파일:** `lib/seo/schema.ts:37, 91, 115`

`generateArticleSchema`, `generateOrganizationSchema`, `generateLocalBusinessSchema` 모두 `${url}/logo.png`을 참조하지만 파일 미존재. Google Rich Results Test에서 오류 발생.

**수정:** `public/logo.png` 추가하거나, 기존 `public/icon.png` (192x192)으로 경로 변경

---

### C-3. 글로벌 레이아웃에 Organization/WebSite 스키마 없음
**심각도:** CRITICAL | **파일:** `app/layout.tsx`

Organization과 WebSite JSON-LD가 홈페이지(`app/page.tsx`)에만 렌더링됨. 블로그 포스트나 상담 페이지에 직접 랜딩하면 Google이 사이트 엔티티를 인식할 수 없음.

**수정:** `app/layout.tsx`에 글로벌 Organization + WebSite 스키마 삽입

---

### C-4. Article 스키마 `image`가 string — `ImageObject` 필요
**심각도:** CRITICAL | **파일:** `lib/seo/schema.ts:22`

Google Article rich result은 `image`가 `ImageObject`(url, width, height 포함)여야 함. 현재 단순 URL string으로 출력되어 rich result 자격 미달.

**수정:**
```ts
image: {
  '@type': 'ImageObject',
  url: post.thumbnail_url,
  width: 1200,
  height: 630,
}
```

## HIGH (1주 내 수정)

### H-1. 상담 페이지 canonical URL 전체 누락
**파일:** `app/counseling/couple/page.tsx` 외 7개 상담 상세 페이지

모든 상담 페이지가 인라인 `metadata` 객체를 사용하며 `alternates.canonical`이 없음. `generatePageMetadata()`를 사용하면 자동으로 canonical 포함됨.

---

### H-2. AI 크롤러 전면 차단
**파일:** `app/robots.ts:13-17`

GPTBot, ChatGPT-User, PerplexityBot 차단으로 ChatGPT 검색, Perplexity AI 노출 0%.

**권장:** 블로그/상담 콘텐츠는 AI 검색 크롤러에 허용, 학습용 크롤러(CCBot)만 차단:
```ts
rules: [
  { userAgent: '*', allow: '/', disallow: ['/api/', '/actions/'] },
  { userAgent: 'CCBot', disallow: '/' },
  { userAgent: 'Bytespider', disallow: '/' },
  // GPTBot, ChatGPT-User, PerplexityBot → 삭제 (허용)
],
```

### H-3. `llms.txt` 파일 없음
AI 검색 엔진의 사이트 구조 이해를 위한 `llms.txt` 파일이 없음.

**수정:** `public/llms.txt` 생성 — 사이트 소개, 서비스, 카테고리, 팀 정보 포함

---

### H-4. `export const viewport` 누락
**파일:** `app/layout.tsx`

Next.js App Router에서 viewport는 별도 export 필요. 현재 누락 시 `<meta name="viewport">` 태그가 빠질 수 있어 모바일 인덱싱에 영향.

**수정:**
```ts
import type { Viewport } from 'next';
export const viewport: Viewport = {
  width: 'device-width',
  initialScale: 1,
  themeColor: '#2d6a4f',
};
```

## 잘하고 있는 점

- **SSR/SSG 렌더링**: 모든 콘텐츠가 서버 렌더링되어 JS 없이도 크롤링 가능
- **보안 헤더**: HSTS, X-Frame-Options, Referrer-Policy 등 유지 (CSP는 외부 서비스 연동 충돌로 제거)
- **스키마 커버리지**: Organization, LocalBusiness, WebSite, BlogPosting, BreadcrumbList, FAQPage, Person, Course, Service 8종 구현
- **next/image 사용**: 12개 컴포넌트에서 Image 최적화 활용, AVIF/WebP 포맷 설정
- **폰트 전략**: `display: 'swap'` 설정으로 렌더 블로킹 방지
- **접근성**: skip-to-content 링크, `lang="ko"`, aria 속성 활용
- **CTA 전환 퍼널**: 블로그 → 상담 프로그램 → 예약으로 이어지는 구조적 내부 링크
- **FAQ 스키마**: 모든 상담 페이지 + 연락처에 FAQPage 구조화 데이터
